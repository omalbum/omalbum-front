## Build and Run on Production Server
1. Tag a commit as a release version with the following rule:
``` vM.m.p ``` being M: Major version, m: minor version and p: patch
2. Push tag:
``` git push --tag ```

Example:
1. ``` git tag v1.0.2 ```
2. ``` git push --tag ```
